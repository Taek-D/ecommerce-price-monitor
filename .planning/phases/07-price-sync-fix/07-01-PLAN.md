    **Bug 1 — Persistent state (`_sourcing_price_state`):**

    The root cause is that `_sourcing_price_state` (line 1919) is a plain dict that resets on every restart. After restart, the first run at lines 2991-2993 always hits `prev_price is None`, stores state, and returns — meaning the FIRST real price change after any restart is silently missed.

    Fix: Add file-backed persistence for `_sourcing_price_state`, following the exact same atomic write pattern already used by `save_state()` in `musinsa_price_watch.py` (tmp file + `os.replace`).

    1. Define `_SOURCING_PRICE_STATE_FILE = "sourcing_price_state.json"` near line 1919
    2. Add `_load_sourcing_price_state() -> dict[int, int]` — reads JSON file, returns `{}` if file missing or corrupt. Keys in JSON are strings, convert to int on load.
    3. Add `_save_sourcing_price_state(state: dict[int, int])` — atomic write via tmp + `os.replace` (same pattern as `save_state()` in musinsa_price_watch.py)
    4. At the START of `sync_price_from_sourcing()` (after line 2725), load persisted state: `_sourcing_price_state.update(_load_sourcing_price_state())` — only load if `_sourcing_price_state` is empty (first call after restart)
    5. At the END of `sync_price_from_sourcing()` (before the Discord notification section around line 3104), save state: `_save_sourcing_price_state(_sourcing_price_state)`

    **Bug 2 — First-run skip is correct but needs state file awareness:**

    The `if prev_price is None: store and continue` logic at lines 2991-2993 is actually correct behavior for truly new rows. With the persistence fix above, this path only triggers for genuinely new rows (not seen in any previous run), which is the desired behavior.

    **Important: Do NOT change the `new_price <= current_sale_price` condition (line 3028).** Per REQUIREMENTS.md "Out of Scope" section, the policy is "상향만 지원" (upward only). The condition is correct for current policy — the bug is that the function never reaches this line due to Bug 1.

    Write tests in `tests/test_price_sync.py`:
    - Test `_load_sourcing_price_state` with valid file, missing file, corrupt file
    - Test `_save_sourcing_price_state` creates valid JSON, atomic write
    - Test the detection logic: mock sheet data + state file, verify `update_sale_price` is called
    - Use `unittest.mock.patch` and `AsyncMock` patterns matching existing test style in `test_coupang_utils.py` and `test_sourcing_tab.py`
