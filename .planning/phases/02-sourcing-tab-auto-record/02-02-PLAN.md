1. In `coupang_manager.py`, add `async def _record_order_to_sourcing_tab(...)` function. Place it after `_load_sourcing_info_by_vid` and `_resolve_sourcing_tab_name` (in the sourcing section, around line ~1630 area, before the sync functions).

Function signature and logic:
```python
async def _record_order_to_sourcing_tab(
    sh,  # gspread Spreadsheet object (already opened, reuse from process_new_orders)
    sourcing_info_by_vid: dict[str, dict],
    *,
    order_id: str,
    vendor_item_id: str,
    buyer_name: str,
    product_name: str,
    qty: int,
    paid_total: int | None,
) -> None:
```

Logic flow:
a. Look up `vendor_item_id` in `sourcing_info_by_vid`. If not found:
   - Log warning: `f"소싱탭 기록 스킵: vendorItemId 매핑 없음 | orderId={order_id} vid={vendor_item_id}"`
   - Send Discord embed via `post_webhook(COUPANG_ORDER_WEBHOOK, ...)` with title "소싱탭 기록 실패", color=15158332 (red), fields: 주문ID, vendorItemId, 상품명, 사유="소싱목록에 vendorItemId 매핑 없음"
   - Return early

b. Get `url` and `buy_price` from sourcing info dict.

c. Call `_resolve_sourcing_tab_name(url)` to get `tab_name`. If None:
   - Log warning: `f"소싱탭 기록 스킵: 도메인 매핑 없음 | orderId={order_id} url={url}"`
   - Send Discord embed with title "소싱탭 기록 실패", color=15158332, fields: 주문ID, vendorItemId, 구매링크=url, 사유="URL 도메인에 대응하는 소싱처 탭 없음"
   - Return early

d. Try to open tab: `ws = sh.worksheet(tab_name)`. Catch `gspread.exceptions.WorksheetNotFound`:
   - Log warning: `f"소싱탭 기록 스킵: 탭 없음 | orderId={order_id} tab={tab_name}"`
   - Send Discord embed with title "소싱탭 기록 실패", fields: 주문ID, 소싱처탭=tab_name, 사유="스프레드시트에 탭 없음"
   - Return early

e. Build row list matching sourcing tab column structure. The row must be a list where index positions correspond to columns A through M:
   - A (index 0): "" (구매날짜 — out of scope per REQUIREMENTS, leave empty)
   - B (index 1): buyer_name (주문자명)
   - C (index 2): "" (수취인명)
   - D (index 3): "" (안심번호)
   - E (index 4): "" (배송지)
   - F (index 5): "" (메모)
   - G (index 6): product_name (상품명)
   - H (index 7): str(qty) (수량)
   - I (index 8): url (구매처 URL)
   - J (index 9): "" (배송회사)
   - K (index 10): "" (empty)
   - L (index 11): str(paid_total) if paid_total else "" (판매가격 — 쿠팡 결제금액)
   - M (index 12): str(buy_price) if buy_price else "" (매입가격 — 소싱목록 H열)

f. Call `ws.append_row(row, value_input_option="USER_ENTERED")`. Wrap in try/except, log error on failure but do NOT re-raise (non-blocking).

g. Log success: `f"소싱탭 기록 완료: {tab_name} | orderId={order_id} | {product_name}"`
